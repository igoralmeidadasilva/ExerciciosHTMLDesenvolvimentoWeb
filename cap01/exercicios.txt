Exercício 1.1 Quais são os 3 tipos de códigos básicos de uma página de hipertexto? Explique para
que serve cada um dos tipos.
R:HTML, CSS e Javascript.
O Html é responsável por montar o design do site.
O Css estilisa o site.
O Javascript adiciona eventos e comportamentos aos componentes do site.


Exercício 1.2 Quantos arquivos CSS e Javascript podem estar vinculados a uma mesma página de
hipertexto? Justifique sua resposta. 
R: Vários arquivos CSS e javascript podem estar vinculadas a uma única página HTML, isso é uma prática comum e recomendavel uma vez que a modularizar os arquivos é uma boa prática. Porém é importante levar em consideração o desenpenho do site uma vez que muitos arquivos exigem mais carregamento.


Exercício 1.3 É possível diferentes páginas de hipertexto terem vínculo com os mesmos arquivos
Javascript e CSS? Justifique sua resposta.
R: Sim é possivel, arquivos HTML tem podem compartilhar o mesmo arquivo CSS e javascript, uma vez carregado o arquivo ele fica na memória cache do navegador e não precisa ser baixado de novo.

Exercício 1.4 Descreva a diferença entre uma página de hipertexto estática e uma dinâmica,
pontuando as vantagens e desvantagens de cada um dos dois tipos.
R: Uma página de hipertexto estática são arquivos prontos, que simplesmente serão baixados do servidor web que hospeda o site. Já uma página dinâmica é gerada a partir de um programa especial chamado de aplicação web. Um site estático tem a característica de não demandar atualizações com muita frequência e são criados e raramente alterados durante a vida de um site, já uma aplicação web é gerada dinamicamente pelo servidor (normalmente aplicações web são compostas por páginas geradas pelo servidor e páginas estáticas).


Exercício 1.5 Qual é o caminho percorrido por uma requisição HTTP a uma página de hipertexto estática?
R:  1. Tudo começa com o usuário informando uma URL na barra de endereços do navegador.
    2. O navegador acessa o servidor DNS e recupera o IP do servidor.
    3. Uma vez que o endereço IP do servidor web tenha sido obtido, uma requisição HTTP é feita ao servidor web correspondente ao IP em questão.
    4. O Servidor então, retornar os arquivos que compõe o site.
    5. O navegador recebe a resposta do servidor e renderiza para o usuário o html.
    6. O navegador aplica o CSS.
    7. E por último o navegador aplica o Javascript.
